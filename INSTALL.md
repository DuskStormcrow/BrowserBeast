# BrowserBeast‚ĄĘ v1.0.1 Install Guide

BrowserBeast‚ĄĘ v1.0.1 Freeware may be distributed on its own or bundled as a
companion extension inside the DesktopBeast‚ĄĘ v1.0 EA RC2 ZIP. BrowserBeast must
still be loaded separately as a browser extension.

## Chrome / Chromium Sideload

1. Unzip the BrowserBeast release package, or extract the DesktopBeast ZIP and
   open the included BrowserBeast folder.
2. Open `chrome://extensions`.
3. Enable **Developer mode**.
4. Click **Load unpacked**.
5. Select the unzipped `BrowserBeast-v1.0.1-Freeware` folder.
6. Pin BrowserBeast to your toolbar if desired.

## First Test

1. Open a normal webpage.
2. Click the BrowserBeast toolbar icon.
3. Choose **Full Render**.
4. Confirm the preview appears.
5. Try **Open Full Review**, **Copy Markdown**, and **Download .md**.

## Highlighted Text Test

1. Select text on a webpage.
2. Open BrowserBeast.
3. Choose **Highlighted Text**.
4. Confirm only the selected text is captured.

## Notes

BrowserBeast‚ĄĘ cannot capture browser-internal pages such as `chrome://extensions`, `edge://settings`, or `about:blank`.

Chrome may warn that BrowserBeast‚ĄĘ can read and change data on websites. BrowserBeast‚ĄĘ requests broad access so it can capture the page you choose, wherever you are. Capture only runs when you click the BrowserBeast‚ĄĘ toolbar button.

BrowserBeast captures browser/web artifacts. DesktopBeast captures desktop/application/window artifacts. ArchiveBeast‚ĄĘ Library can review BrowserBeast Markdown captures when the Browser shelf is pointed at the folder containing those captures. BrowserBeast does not automatically sync into ArchiveBeast yet.

Review captures before sharing. BrowserBeast‚ĄĘ may preserve private/logged-in indicators or page content visible at capture time.
